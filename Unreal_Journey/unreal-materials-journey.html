Foundation
Understanding UE5 Materials

Materials define how every surface in your scene looks and responds to light. Master these fundamentals before building complex shaders.

00

What Are Materials in UE5?

Materials control how surfaces look — their colour, shininess, roughness, and glow. Every object in your scene has a material applied to it. Without materials, everything would appear as a flat grey surface.

Think of a material as paint with superpowers. Regular paint only gives you colour. A UE5 material controls colour, reflectivity, roughness, transparency, glow, and even fake surface bumps — all at once.

Node-Based Editor — Visual, No Code

UE5 uses a node-based Material Editor. Instead of writing shader code, you connect visual nodes with wires. Each node does something simple (pick a colour, sample a texture, multiply two values). Connected together, they create complex surface effects.

  • Nodes — small boxes that hold a value or perform an operation
  • Wires — connections that pass data from one node to another
  • Result Node — the final destination where all your wires end up
No coding required: The Material Editor is entirely visual. You drag, drop, and connect nodes. If you can follow a flowchart, you can build materials.

PBR — Physically Based Rendering

UE5 materials use PBR (Physically Based Rendering). This means materials mimic how light behaves in the real world:

  • Rough surfaces scatter light → matte appearance
  • Smooth surfaces reflect light → shiny/mirror appearance
  • Metals reflect their own colour → gold looks gold, copper looks copper
  • Non-metals reflect white light → plastic, wood, fabric
PBR is like a physics engine for light. Just as a physics engine makes objects fall realistically, PBR makes surfaces look realistic by following real-world rules about how light bounces, scatters, and absorbs.
Milestone: You understand that materials are node-based visual shaders that control surface appearance using physically based rendering.
01

Create Your First Material

Create & Open

  1. In the Content Browser, navigate to your Materials folder
  2. Right-click in empty space → Material
  3. Name it M_MyFirstMaterial
  4. Double-click to open the Material Editor
Naming convention: Always prefix materials with M_ — this keeps your Content Browser organised and is standard UE5 practice.

The Material Graph

The Material Editor opens with a large graph area and one node already placed: the Result Node (also called the Main Material Node). This is where all your node connections end up. It has several input pins:

Add a Colour

  1. Right-click in the graph → search for Constant3Vector
  2. Click to place the node — it appears as a small box with an RGB output
  3. Double-click the node to open the colour picker
  4. Choose any colour you like
  5. Drag a wire from the node's output pin to the Base Color input on the Result Node
  6. The preview sphere in the corner updates immediately
  7. Click Apply (top-left) then Save

Apply to a Mesh

  1. Go back to the main editor (click the Level tab)
  2. Select any mesh in the Viewport (a cube, a sphere, any object)
  3. In the Content Browser, find your material M_MyFirstMaterial
  4. Drag the material from the Content Browser onto the mesh in the Viewport
  5. The mesh updates instantly with your chosen colour
Applying a material is like painting an object. Drag the paint (material) onto the surface (mesh) and it coats it immediately.

02

PBR Properties

Now that you can set a base colour, it is time to control how the surface reacts to light. These three properties — Base Color, Metallic, and Roughness — are the core of PBR materials.

Base Color

The surface colour as an RGB value (red, green, blue). This is the colour you see when light hits the surface. Use a Constant3Vector node for a solid colour, or a TextureSample node for a texture image.

Keep it realistic: Pure black (0, 0, 0) and pure white (1, 1, 1) do not exist in the real world. Use values between 0.02 and 0.95 for believable results.

Metallic

Controls whether the surface is a metal or non-metal.

  • 0 = non-metal (plastic, wood, fabric)
  • 1 = full metal (steel, gold, aluminium)

Use a Constant node (single float) and plug it into the Metallic pin. In practice, this value should almost always be exactly 0 or 1 — not something in between.

Roughness

Controls how smooth or rough the surface is.

  • 0 = perfectly smooth (mirror reflection)
  • 1 = completely rough (matte, diffuse)

Use a Constant node and plug it into the Roughness pin. Unlike Metallic, in-between values are common and encouraged.

Normal Maps

A Normal Map is a special texture that fakes surface detail without adding extra geometry. It makes a flat surface look like it has bumps, grooves, scratches, or pores — all without increasing polygon count.

  • Import a normal map texture into the Content Browser
  • Create a TextureSample node and assign the normal map
  • Plug it into the Normal input on the Result Node
A normal map is like an optical illusion for light. The surface is still flat, but the normal map tells the lighting engine to pretend there are bumps. The result: convincing detail at zero geometry cost.

Add Metallic & Roughness to Your Material

  1. Open your material M_MyFirstMaterial
  2. Right-click in the graph → search for Constant (single value, not Constant3Vector)
  3. Place the node and set its value to 0 (non-metal)
  4. Wire it to the Metallic input
  5. Create another Constant node with a value of 0.4
  6. Wire it to the Roughness input
  7. Click Apply → Save — observe the preview sphere

03

Using Textures

Solid colours are a great start, but real-world surfaces have variation, patterns, and detail. Textures are image files that provide this visual complexity.

Import a Texture

  1. Find a texture image file (PNG, JPG, TGA, or EXR)
  2. Drag it from your file explorer directly into the Content Browser
  3. UE5 imports it automatically — you will see a thumbnail appear
  4. Organise it into a Textures folder for tidiness

TextureSample Node

  1. Open your material in the Material Editor
  2. Right-click in the graph → search for TextureSample
  3. Place the node in the graph
  4. Select the TextureSample node, then in the Details Panel, click the texture dropdown and choose your imported texture
  5. Drag a wire from the RGB output to the Base Color input on the Result Node
  6. Click Apply → the preview sphere now shows your texture

Texture Coordinates (Tiling)

By default, a texture stretches across the entire surface once. To tile (repeat) a texture:

  1. Right-click in the graph → search for TexCoord (Texture Coordinate)
  2. Set UTiling and VTiling to a higher number (e.g. 4, 4)
  3. Wire the TexCoord output into the UVs input on your TextureSample node
  4. The texture now repeats 4 times across each axis
Tip: Higher tiling numbers make the texture repeat more, showing more detail but also making the pattern more visible. Find a balance that looks natural.

Combining Textures

You can blend and layer textures for richer results:

  • Multiply — darkens where both textures overlap. Great for adding dirt or grunge.
  • Lerp (Linear Interpolate) — blends between two textures using a third value (0 = first texture, 1 = second texture). Use a mask or gradient to control the blend.
  • Add — brightens by combining values. Useful for layering light effects.

04

Material Instances

Why Material Instances? Every time you edit a base material, UE5 must recompile the entire shader. This can take seconds or even minutes. A Material Instance lets you change properties (colour, roughness, textures) without recompiling — changes are instant.

A base material is like a recipe. A Material Instance is like making that recipe but swapping one ingredient (e.g. red paint instead of blue). You do not need to rewrite the whole recipe — just change the one value.

Convert Nodes to Parameters

Before creating an instance, you need to turn your constant values into Parameters — exposed values that instances can override.

  1. Open your base material in the Material Editor
  2. Right-click your Constant3Vector node (Base Color) → Convert to Parameter
  3. Name it BaseColor
  4. Do the same for your Roughness Constant → name it Roughness
  5. Do the same for Metallic → name it Metallic
  6. Click Apply → Save
Visual change: Parameter nodes look slightly different from constant nodes — they have a name label and a border highlight. This tells you the value is exposed.

Create a Material Instance

  1. In the Content Browser, right-click your base material
  2. Select Create Material Instance
  3. Name it MI_MyFirstMaterial_Red (use the MI_ prefix)
  4. Double-click to open the Material Instance Editor
  5. You will see all your parameters listed with checkboxes
  6. Tick the BaseColor checkbox to enable it
  7. Change the colour to red — the preview updates instantly
  8. Tick Roughness and adjust the slider
  9. Save the instance

Use Case: One Material, Many Variations

Imagine a brick wall material. Instead of creating separate materials for red bricks, grey bricks, and white bricks, you create:

  • M_Brick — the base material with exposed parameters
  • MI_Brick_Red — instance with red base colour
  • MI_Brick_Grey — instance with grey base colour, slightly higher roughness
  • MI_Brick_White — instance with white base colour

All three instances share the same shader logic. If you change the node graph in M_Brick, all instances update automatically.

Milestone: You can convert constants to parameters, create Material Instances, and adjust instance properties without recompiling the shader.
05

Emissive Materials (Glow)

Emissive materials glow — they appear to emit light from their surface. This is perfect for neon signs, screens, magical effects, sci-fi panels, and any surface that should look self-illuminated.

How Emissive Works

  • The Emissive Color input on the Result Node controls self-illumination
  • Any colour plugged in here will appear to glow
  • To make it actually bloom (visible glow halo), multiply the colour by a value greater than 1
  • Higher multiplier = stronger glow (try values like 5, 10, 20, or even 100)

Practical Example: Glowing Panel

  1. Create a new material: M_GlowPanel
  2. Add a Constant3Vector → pick a bright cyan colour
  3. Add a Multiply node (right-click → search "Multiply")
  4. Wire the colour into the A input of Multiply
  5. Add a Constant node, set its value to 15
  6. Wire the Constant into the B input of Multiply
  7. Wire the Multiply output into Emissive Color on the Result Node
  8. Set Roughness to 0.1 (smooth surface)
  9. Click Apply → Save

Bloom Must Be Enabled

For the glow halo to be visible, your scene needs Bloom enabled in the Post Process Volume:

  1. Select your Post Process Volume in the Outliner (or create one: Place Actors → Visual Effects → Post Process Volume)
  2. In the Details Panel, find Bloom under Lens
  3. Tick the checkbox to enable it
  4. Set Intensity to around 1.0 (default is usually fine)
No glow visible? Check that Bloom is enabled in your Post Process Volume and that the volume's bounds are set to "Infinite Extent (Unbound)" so it affects the entire level.

Great Uses for Emissive Materials

  • Neon signs and LED strips
  • Computer screens and holographic displays
  • Magical effects and energy fields
  • Sci-fi panels and control consoles
  • Glowing crystals and fantasy elements
  • Emergency lighting and exit signs
Milestone: You can create emissive materials with controllable glow intensity using a Multiply node. You know that Bloom must be enabled for the glow halo to appear.
06

Decals & Special Effects

Decals are materials projected onto surfaces. Instead of replacing the surface material, a decal overlays on top. Use them for dirt, cracks, graffiti, signs, puddles, scorch marks — anything that sits on top of existing surfaces.

Decals are like stickers. You place them on top of a surface without changing the underlying material. Multiple decals can overlap, just like layering stickers.

Create a Deferred Decal Material

  1. Create a new material: M_Decal_Cracks
  2. Open the Material Editor
  3. In the Details Panel (left side), find Material Domain
  4. Change it from Surface to Deferred Decal
  5. Set Blend Mode to Translucent
  6. Set Decal Blend Mode to Translucent (or DBuffer Translucent Color, Normal, Roughness for full PBR decals)
  7. Add a TextureSample node with your decal texture
  8. Wire the RGB output to Base Color
  9. Wire the Alpha output to Opacity (this controls the shape of the decal)
  10. Click Apply → Save

Place a Decal in the Scene

  1. Go to Place Actors panel
  2. Search for Decal (under Visual Effects)
  3. Drag a Decal Actor into the Viewport
  4. In the Details Panel, assign your decal material (M_Decal_Cracks)
  5. Position the decal near a wall or floor — it projects onto nearby surfaces
  6. Use the Decal Size property to control how large the projection area is
  7. Rotate the decal to align with the surface
Tip: The decal projects along its local X axis (the blue arrow). Point the blue arrow at the surface you want to project onto.

Opacity Masks for Shaped Decals

Most decal textures need an alpha channel (transparency) so they are not just a square projected onto the surface. The alpha channel defines the shape:

  • White in the alpha = fully visible
  • Black in the alpha = fully transparent
  • Grey in the alpha = partially transparent

If your texture does not have an alpha channel, create a mask texture separately and use it for the Opacity input.

Common Decal Uses

  • Cracks and damage on walls and floors
  • Dirt, mud, and weathering effects
  • Signage, graffiti, and posters
  • Puddles and water stains
  • Scorch marks and impact craters
  • Blood splatters (for game environments)
Milestone: You can create Deferred Decal materials with opacity masks and place them in your scene to add surface detail without modifying the underlying mesh.

Tips & Best Practices

Naming Conventions

Asset TypePrefixExample
MaterialM_M_BrickWall
Material InstanceMI_MI_BrickWall_Red
TextureT_T_BrickWall_BC (Base Color)
Normal MapT_T_BrickWall_N
Roughness MapT_T_BrickWall_R
Why bother? Consistent naming lets you find any asset instantly. In large projects, you might have hundreds of materials — prefixes keep them searchable and sorted.

Performance

  • Use Material Instances wherever possible. They are cheaper to modify and faster to preview than editing base materials.
  • Avoid overly complex node graphs. Each node adds to the shader instruction count. Keep it simple unless you need complexity.
  • Share base materials. Create a few versatile base materials with parameters, then use instances for variations.
  • Texture resolution matters. Use 2048x2048 for hero surfaces and 1024x1024 or smaller for background elements. Larger textures consume more memory.

Keep Node Graphs Clean

  • Press C with nodes selected to create a Comment Box around them — label groups of nodes for clarity
  • Use Reroute Nodes (double-click a wire) to keep wires tidy
  • Align nodes with the grid — it makes the graph easier to read
  • Delete unused nodes — they add visual clutter even if unconnected

Visual Quality Tips

  • Roughness variation makes scenes look real. In the real world, no surface has perfectly uniform roughness. Add subtle roughness variation via textures or noise.
  • Mix materials on a single mesh using Material Slots. Many meshes support multiple material IDs — assign different materials to different faces.
  • Use detail normal maps for close-up surfaces. Layer a micro-detail normal (fabric weave, metal grain) on top of the main normal map using a BlendAngleCorrectNormal node.
  • Always save after editing. The Material Editor does not auto-save. If UE5 crashes, unsaved material changes are lost.
Remember: Click Apply first to see changes in the Viewport, then Save (Ctrl+S) to write to disk. Apply without Save means your changes are visible but not persisted.
