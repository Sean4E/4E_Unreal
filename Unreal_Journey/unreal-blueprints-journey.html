Foundation
Understanding Blueprints

Before you wire anything, understand what Blueprints are, how they work, and why they are the fastest way to prototype gameplay in UE5.

00

What Are Blueprints?

Blueprints are UE5's visual scripting system. Instead of typing lines of code, you drag colourful nodes onto a graph and connect them with wires. Each node performs a single action — play a sound, move an object, check a condition — and the wires define the order things happen. Every gameplay feature in Unreal Engine 5 can be built entirely with Blueprints.

Think of Blueprints like a flowchart. Each box is an instruction, and the arrows between boxes say "do this, then do that." The engine reads your flowchart from left to right and follows every connection.

Two Main Types

TypeWhat It DoesWhen to Use
Level BlueprintLives inside a single Level. Controls events that are specific to that map.Triggering a cutscene when the player enters a room. One-off level logic.
Actor BlueprintA reusable object you create once and drag into any Level as many times as you like.A door that opens, a pickup item, a light switch — anything you want to reuse.
Rule of thumb: If you need the behaviour in more than one place, make it an Actor Blueprint. If it only happens once in this specific level, the Level Blueprint is fine.

Key Concepts

  • Events — the starting points of your logic. Something happens (the game starts, the player overlaps a trigger) and an event fires.
  • Functions — reusable groups of nodes that perform a task. Call them from anywhere in the same Blueprint.
  • Variables — named containers that store values (a number, a true/false, a colour). Your Blueprint can read and write these at any time.
  • Flow Control — nodes that make decisions. Branch (if/else), Flip Flop (toggle), For Each Loop, and more.

Blueprint Pin Colours

Every pin on a Blueprint node is colour-coded by its data type. Learn these colours and you will always know what kind of data a wire carries:

Execution (white)
Boolean (green)
Float (yellow-green)
Integer (cyan)
String (magenta)
Object Ref (blue)
Vector (gold)
Tip: You can only connect pins of the same colour. If you try to connect a green Boolean pin to a yellow Float pin, UE5 will refuse — or offer an automatic conversion node.
Milestone: You understand that Blueprints are visual flowcharts, there are two main types (Level and Actor), and you can name the four key concepts: Events, Functions, Variables, Flow Control.
01

Your First Blueprint Actor

Time to create your very first reusable Blueprint. You will build an Actor that contains a glowing cube with a light attached to it.

  1. In the Content Browser, right-click in an empty area
  2. Select Blueprint Class
  3. Choose Actor as the parent class
  4. Name it BP_MyFirstActor
  5. Double-click to open the Blueprint Editor

The Blueprint Editor

When you open a Blueprint, you see three key areas:

  • Components Panel (top-left) — lists all the parts attached to this Actor
  • Viewport (centre) — a 3D preview of your Actor and its components
  • Event Graph (tab at top) — the node graph where you add logic
The Components panel is like a parts list. The Viewport shows how those parts look assembled. The Event Graph is where you tell those parts what to do.

Add Components

  1. In the Components panel, click + Add
  2. Search for Static Mesh and add it
  3. In the Details Panel, set the Static Mesh to a Cube (search for "Cube" in the mesh dropdown)
  4. Click + Add again
  5. Search for Point Light and add it
  6. Drag the Point Light slightly above the cube in the Viewport
  7. Set the light colour to orange in the Details Panel for a warm glow

02

The Event Graph

The Event Graph is where your Blueprint comes alive. Every Actor Blueprint has an Event Graph tab. Open it and you will see a dark canvas with a grid — this is where you place nodes and connect them with wires.

The Two Most Important Events

EventWhen It FiresUse For
Event BeginPlayOnce, when the game starts (or when this Actor is spawned)Setup logic: set initial values, play intro sounds, enable/disable things
Event TickEvery single frame (60+ times per second)Continuous checks. Use sparingly — it can hurt performance.
Performance warning: Never put heavy logic on Event Tick. If you only need to react to something happening, use events and overlaps instead. Tick should be a last resort.

Your First Node Chain: Hello Unreal!

  1. Open BP_MyFirstActor and switch to the Event Graph tab
  2. You should see Event BeginPlay already on the graph (red header)
  3. Drag from the white execution pin (right side of BeginPlay) into empty space
  4. A search menu appears — type Print String
  5. Click Print String to add it
  6. In the Print String node, change the text to Hello Unreal!
  7. Compile and Save
  8. Back in the main editor, press Alt + P to Play in Editor
  9. Look at the top-left of the Viewport — you should see "Hello Unreal!" printed on screen

03

Variables

Variables are named containers that hold values. Think of them as labelled boxes: the label says what the box is for, and the contents can change over time. Your Blueprint reads and writes these values to track state — is a door open? How much health does the player have? What colour is the light?

Variable Types

TypePin ColourWhat It StoresExample
Boolean● GreenTrue or FalseisOpen = false
Float● Yellow-greenDecimal numberSpeed = 3.5
Integer● CyanWhole numberScore = 42
String● MagentaTextPlayerName = "Alex"
Vector● GoldX, Y, Z coordinatesSpawnPoint = (100, 0, 50)

Create a Variable

  1. Open your Blueprint and look at the My Blueprint panel (bottom-left)
  2. Under the Variables section, click the + button
  3. A new variable appears — name it isOpen
  4. In the Details Panel on the right, set the Variable Type to Boolean
  5. Set the Default Value to false (unticked checkbox)
  6. Compile to save the default value

Get vs Set

When you drag a variable onto the Event Graph, UE5 asks you to choose:

  • Get — reads the current value (used as input to other nodes)
  • Set — writes a new value (changes the stored data)
Get is like reading the label on a jar to see what is inside. Set is like putting something new into the jar.

04

Open a Door (Interactive Blueprint)

Now you will build something real: a door that swings open when the player walks near it. This Blueprint combines everything you have learned so far — components, events, variables, and flow control.

Create BP_Door

  1. Content Browser → right-click → Blueprint Class → Actor
  2. Name it BP_Door
  3. Open it and add a Static Mesh component — set the mesh to a door shape (or use the Starter Content door, or a scaled cube as a placeholder)
  4. Add a Box Collision component — this is the trigger zone
  5. Scale the Box Collision so it extends in front of the door (the area the player will walk through)
  6. Select the Box Collision, go to Details → Collision section → set Collision Presets to OverlapAllDynamic

Add a Timeline for Smooth Rotation

A Timeline node creates smooth animation by outputting a value that changes over time (like 0 to 1 over 1 second).

  1. Switch to the Event Graph
  2. Right-click → search for Add Timeline → name it DoorTimeline
  3. Double-click the Timeline to open it
  4. Click + Float Track → name it DoorAlpha
  5. Add two keyframes: (Time: 0, Value: 0) and (Time: 1.0, Value: 1.0)
  6. Set the Length to 1.0 second
  7. Close the Timeline editor

Wire Up the Logic

  1. Add Event ActorBeginOverlap (right-click → search → or look under Events)
  2. Add a Flip Flop node (so the door opens on first overlap, closes on second)
  3. Connect: BeginOverlap exec → Flip Flop exec
  4. Flip Flop A output → Timeline Play from Start
  5. Flip Flop B output → Timeline Reverse from End
  6. Add a Lerp (Rotator) node
  7. Set Lerp A to (0, 0, 0) (door closed) and B to (0, 0, 90) (door open)
  8. Connect the Timeline DoorAlpha output to the Lerp Alpha input
  9. Add Set Relative Rotation node — target is the DoorMesh
  10. Connect: Timeline Update exec → Set Relative Rotation exec
  11. Connect: Lerp Return Value → Set Relative Rotation → New Rotation

Test It

  1. Compile and Save
  2. Drag BP_Door into your level
  3. Press Alt + P to play
  4. Walk your character into the trigger zone
  5. The door should smoothly swing open
  6. Walk in again — the door should close (thanks to Flip Flop)
Common mistake: If the door does not rotate, make sure the DoorMesh pivot point is at the hinge edge, not the centre. You may need to adjust the mesh position relative to the root component.

05

Toggle a Light

Build a light that turns on and off when the player enters a trigger zone. This introduces the Branch node — Blueprint's version of an if/else statement.

Create BP_ToggleLight

  1. Content Browser → right-click → Blueprint Class → Actor
  2. Name it BP_ToggleLight
  3. Add a Point Light component
  4. Add a Box Collision component (trigger zone)
  5. Scale the Box Collision to cover the interaction area
  6. Set collision preset to OverlapAllDynamic

Add the Variable

  1. In the My Blueprint panel, click + under Variables
  2. Name it isOn, type Boolean, default true (light starts on)
  3. Compile to lock in the default

Wire Up the Logic

  1. Add Event ActorBeginOverlap
  2. Add a Branch node (right-click → search "Branch")
  3. Drag isOn as a Get and connect it to the Branch Condition pin
  4. True path: add Set Visibility on the Point Light → set to false (turn off)
  5. Then add Set isOn → set to false
  6. False path: add Set Visibility on the Point Light → set to true (turn on)
  7. Then add Set isOn → set to true
Cleaner alternative: Replace the Branch with a Flip Flop node and a single Toggle Visibility call. Both approaches work — use whichever makes more sense to you.

06

Trigger Events

Blueprints can trigger all kinds of effects: sounds, particles, camera shakes, and object movement. Here are practical examples you can try in your own projects.

Play a Sound

Use the Play Sound at Location node to trigger audio at a specific point in 3D space.

  1. Right-click → search Play Sound at Location
  2. Set the Sound pin to a Sound Cue or Sound Wave asset
  3. Connect Get Actor Location to the Location pin
  4. Wire the exec pin from an event (e.g. BeginOverlap)

Spawn Particles

Use Spawn Emitter at Location to create a burst of particles.

  1. Right-click → search Spawn Emitter at Location
  2. Set the Emitter Template to a Particle System asset
  3. Connect Get Actor Location for the spawn point
  4. Wire from your trigger event
Niagara: For UE5's newer particle system, use Spawn System at Location instead. Niagara is more powerful but works the same way in Blueprints.

Smooth Object Movement

Combine a Timeline with a Lerp (Vector) to smoothly move any object from point A to point B.

  1. Create a Timeline with a float track (0 to 1)
  2. Add a Lerp (Vector) node
  3. Set A = start position, B = end position
  4. Connect Timeline output to Lerp Alpha
  5. Connect Lerp result to Set Actor Location

Camera Shake

Add impact and drama with the Play World Camera Shake node.

  1. First, create a Camera Shake asset in the Content Browser (right-click → Blueprints → Camera Shake)
  2. Set shake parameters: duration, amplitude, frequency
  3. In your Blueprint, add Play World Camera Shake
  4. Set the Shake pin to your Camera Shake asset
  5. Wire from a trigger event

07

Communication Between Blueprints

So far, each Blueprint has worked on its own. But real gameplay needs Blueprints to talk to each other. A switch in one room should control a light in another. A pickup should update the player's score. Here are three ways to make that happen.

Method 1: Casting

Casting lets you get a reference to a specific type of Blueprint and access its variables and functions.

  1. Get a reference to the other Actor (e.g. from an Overlap event's Other Actor pin)
  2. Add a Cast To node — choose the target Blueprint class (e.g. Cast to BP_ToggleLight)
  3. If the cast succeeds, the As output pin gives you access to that Blueprint's variables and functions
  4. Drag from the As pin and call any function or set any variable on the other Blueprint
Casting is like asking someone: "Are you a BP_ToggleLight?" If they say yes, you can then tell them to do things. If they say no, the cast fails and nothing happens.

Method 2: Get All Actors of Class

When you need to find a Blueprint that is already placed in the level but you do not have a direct reference to it:

  1. Add Get All Actors of Class node
  2. Set the Actor Class to the Blueprint you want to find (e.g. BP_ToggleLight)
  3. The output is an array of all instances of that class in the level
  4. Use Get (a copy) with index 0 to get the first one, or use a For Each Loop to affect them all
Performance note: Get All Actors of Class searches the entire level every time it runs. Do not call it on Tick. Call it once on BeginPlay and store the result in a variable.

Method 3: Custom Events & Event Dispatchers

Custom Events let you define your own event nodes that you can call from other Blueprints.

  • In BP_ToggleLight, right-click the graph → Add Custom Event → name it ToggleMyLight
  • Wire the ToggleMyLight event to your toggle logic
  • From another Blueprint (e.g. BP_Switch), get a reference to BP_ToggleLight and call ToggleMyLight

Event Dispatchers go one step further — they broadcast an event that any number of other Blueprints can listen to.

  • In the sending Blueprint, create an Event Dispatcher in the My Blueprint panel
  • Call it when something happens (e.g. OnSwitchFlipped)
  • In the receiving Blueprint, Bind Event to that dispatcher
  • When the dispatcher fires, all bound Blueprints react

Practical Example: Switch Controls a Light

A wall switch in Room A controls a ceiling light in Room B:

  1. Create BP_Switch with a mesh, a Box Collision, and an Event Dispatcher called OnSwitchToggled
  2. When the player overlaps the switch, Call OnSwitchToggled
  3. Create BP_CeilingLight with a Point Light
  4. In BP_CeilingLight's BeginPlay, use Get All Actors of Class to find BP_Switch
  5. Bind Event to OnSwitchToggled on the switch
  6. When the event fires, toggle the light visibility
Result: The switch and the light are completely separate Blueprints, but they communicate through the Event Dispatcher. You can place them in different rooms, different levels — they stay connected.

Final Checklist & Tips

Before You Submit

  • Always Compile before testing — uncompiled Blueprints will not reflect your latest changes
  • Use Comments (press C) to label groups of nodes — future you will thank present you
  • Name variables clearly: isOpen, doorSpeed, lightIntensity — not var1, temp, x
  • Avoid Event Tick when possible — use overlap events, timers, and dispatchers instead
  • Keep Blueprints modular — one Blueprint per behaviour, communication through events
  • Test frequently — compile and play after every few nodes, not after building 50 nodes at once
  • Save often — UE5 does not auto-save Blueprints

Common Mistakes to Avoid

MistakeWhat HappensFix
Forgetting to CompileOld logic runs instead of your new changesAlways click Compile (or Ctrl+F7) before Play
Disconnected exec wiresNodes exist but never executeMake sure every node is connected to the white execution chain
Heavy logic on TickFrame rate drops, game stuttersMove logic to events, timers, or overlap triggers
Unnamed variablesYou forget what var1 does two days laterUse descriptive names: isLightOn, doorAngle
