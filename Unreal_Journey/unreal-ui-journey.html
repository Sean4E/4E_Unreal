User Interface in UE5

Understand UMG (Unreal Motion Graphics), the visual UI editor, and the difference between screen-space and world-space interfaces.

00

UI in Unreal Engine 5

What is UMG? UMG (Unreal Motion Graphics) is Unreal Engine’s built-in UI framework. It provides a visual Widget Blueprint editor where you design interfaces by dragging and dropping widgets onto a canvas — no code required for layout. Behind the scenes, UMG renders your UI as Slate widgets, UE5’s powerful C++ UI layer.

Think of UMG as a drag-and-drop design tool (like Figma) that lives inside Unreal Engine. You design visually, then wire up logic with Blueprints.

Widget Blueprints — The Visual Editor

A Widget Blueprint is a special Blueprint asset that contains two tabs:

TabPurpose
DesignerVisual canvas where you place and arrange UI widgets (text, buttons, images, etc.)
GraphBlueprint Event Graph where you write the logic behind UI interactions (button clicks, value changes)
Key panels in the Designer: Palette (widget library), Hierarchy (widget tree), Canvas (visual preview), and Details (properties for the selected widget).

Screen-Space vs World-Space

ModeDescriptionUse Cases
Screen-Space (HUD)UI overlays the camera — always faces the player, fixed to screenHealth bars, score, menus, inventory
World-Space (3D)UI exists as a 3D surface in the game worldFloating labels, door keypads, VR/AR panels, enemy health bars
Screen-space UI is like a sticker on your screen. World-space UI is like a sign hanging in the room — you can walk around it, and it has a physical position in 3D space.

Why UI Matters

  • Menus — main menu, settings, character select, loading screens
  • HUDs — health, ammo, minimap, objectives, notifications
  • Inventory & Shops — item grids, drag-and-drop, tooltips
  • XR Interfaces — VR/AR panels that players interact with via controllers or hand tracking
  • Narrative — dialogue boxes, subtitles, quest logs
Good UI is invisible. Players should focus on the experience, not struggle with the interface. Clear hierarchy, consistent styling, and responsive feedback make all the difference.
Milestone: You understand what UMG is, how Widget Blueprints work, and the difference between screen-space and world-space UI.

Building Your First Layout

  1. From the Palette, drag a Canvas Panel onto the canvas — this is your root container
  2. Drag a Text widget onto the Canvas Panel
  3. In the Details panel, set the text to Hello UE5!
  4. Set the Font Size to 48
  5. Set the Anchors to center (the medallion preset in the Anchors dropdown)
  6. Adjust Alignment to 0.5, 0.5 so the text is perfectly centered

Understanding Anchors

Anchors define where a widget is pinned relative to its parent. When the screen resizes, anchored widgets stay in the correct position.

AnchorBehaviour
Top-LeftWidget stays fixed to the top-left corner
CenterWidget stays centered on screen regardless of resolution
StretchWidget stretches to fill all (or part of) the parent — great for backgrounds
Common mistake: Forgetting to set anchors. Your UI may look perfect at 1920×1080 but break at other resolutions. Always set anchors for every widget.

Displaying the Widget on Screen

Adding a Button

  1. From the Palette, drag a Button widget onto your Canvas Panel

Handling the Click Event

  1. Select the button in the Designer
  2. In the Details panel, scroll to Events
  3. Click the green + next to On Clicked — this opens the Graph tab
  4. From the OnClicked event node, drag out and add a Print String node
  5. Type Button Clicked! in the string field
  6. Compile and play — click the button and check the output log

Layout Containers

Layout widgets automatically arrange their children — no manual positioning needed.

ContainerBehaviourUse Case
Canvas PanelFree-form placement with anchorsRoot container, absolute positioning
Horizontal BoxChildren arranged left to rightToolbar buttons, stat row
Vertical BoxChildren arranged top to bottomMenu options, settings list
Grid PanelRows and columns gridInventory grids, level select
Wrap BoxWraps children to next line when fullTag lists, icon galleries
OverlayChildren stacked on top of each otherBackground image + text

Polish Widgets

  • Size Box — force a specific width/height on a child widget
  • Spacer — empty space between elements (set width or height)
  • Border — wraps a child with a background brush and padding
  • Scroll Box — scrollable container for long lists
  • Background Blur — blurs the game view behind the widget (great for pause menus)
Layout containers in UMG work like CSS Flexbox. Horizontal Box is flex-direction: row, Vertical Box is flex-direction: column. If you have web experience, the concepts transfer directly.

Binding Variables — Real-Time Updates

To make the health bar and score update dynamically, you bind widget properties to functions:

  1. Select the Progress Bar in the Designer
  2. Next to the Percent property, click Bind → Create Binding
  3. A function is created in the Graph — it returns a float (0.0 to 1.0)
  4. In the function, get the player character, read its Health variable, divide by MaxHealth, and return the result
// Binding function: Get Health Percent
Get Player Character
    ↓
Cast to BP_PlayerCharacter
    ↓
Get Health  /  Get MaxHealth
    ↓
Divide (Health ÷ MaxHealth)
    ↓
Return (float 0–1)
Performance note: Bindings run every frame. For complex UIs, consider using event-driven updates instead — only update the widget when a value actually changes (e.g., via an Event Dispatcher).

Score Text Binding

Repeat the binding process for the score text:

  1. Select the score Text widget
  2. Bind the Text property → Create Binding
  3. In the function, get the player’s score variable
  4. Convert the integer to text using To Text (Int)
  5. Optionally use Format Text to display as Score: {0}
Milestone: You have a HUD with a health bar, score text, and crosshair that update in real-time from gameplay variables.

Animation & Polish

UMG Animations

UMG has a built-in animation timeline editor, similar to keyframing in After Effects. You can animate any widget property over time.

  1. Open your Widget Blueprint (e.g., WBP_MainMenu)
  2. At the bottom of the Designer, find the Animations panel
  3. Click + Animation and name it FadeIn
  4. Select the widget you want to animate (e.g., a Vertical Box containing your menu)
  5. Add a Track for the widget → choose the property (e.g., Render Opacity)
  6. Set a keyframe at time 0.0 with Opacity = 0
  7. Set a keyframe at time 0.5 with Opacity = 1
  8. Preview the animation in the timeline

Animatable Properties

PropertyEffectUse Case
Render OpacityFade in / fade outMenu appears, notification fades
Render Transform → TranslationSlide in from edgePanel slides from left, toast slides up
Render Transform → ScaleGrow / shrinkButton pulse, popup entrance
Color and OpacityColor shiftHighlight change, damage flash

Playing Animations in Blueprints

// Widget Blueprint Graph — Event Construct
Play Animation
    In Animation: FadeIn
    Start at Time: 0.0
    Num Loops: 1
    Play Mode: Forward
Event Construct fires when the widget is created. This is the perfect place to trigger entrance animations — your menu fades in smoothly instead of popping onto the screen.

Button Hover Effects

Create a separate animation for hover feedback:

  1. Create an animation called ButtonHover
  2. Animate the button’s Render Transform → Scale from 1.0 to 1.05 over 0.1 seconds
  3. In the Graph, bind the On Hovered event → Play Animation Forward
  4. Bind On Unhovered event → Play Animation Reverse

Sound Effects on Clicks

// Event OnClicked (any button)
Play Sound 2D
    Sound: SFX_ButtonClick   // import a .wav click sound
    ↓
// ... rest of button logic
Audio feedback is the difference between a UI that feels flat and one that feels professional. A subtle click sound on buttons and a whoosh on menu transitions go a long way.
Milestone: Your menus fade in smoothly, buttons scale on hover, and clicks produce satisfying audio feedback.

World-Space UI & XR

Widget Component — UI in 3D

A Widget Component renders any Widget Blueprint as a 3D surface in the world. Instead of overlaying the screen, the UI exists as a physical panel that players can look at and walk around.

  1. Create or open any Actor Blueprint
  2. In the Components panel, click Add Component → Widget
  3. In the Details panel, set Space to World
  4. Set Widget Class to your Widget Blueprint (e.g., WBP_InfoPanel)
  5. Adjust Draw Size to control the resolution (e.g., 800 × 600)
  6. Scale and position the component in the viewport

Use Cases for World-Space UI

  • Floating name tags — labels above NPCs or players
  • Health bars above enemies — world-space progress bars that face the camera
  • In-world screens — TV monitors, computer terminals, security cameras
  • Door keypads — interactive panels the player approaches and uses
  • Museum labels — information panels next to exhibits in architectural viz
  • VR/AR menus — panels floating in 3D that the user points at with controllers

XR UI: Widget Interaction Component

In VR/AR, the mouse cursor does not exist. Instead, UE5 provides the Widget Interaction Component — a virtual laser pointer that can interact with world-space Widget Components.

  1. On your VR Pawn (or hand controller), add a Widget Interaction component
  2. Attach it to the motion controller so it points forward
  3. The component emits a ray that detects Widget Components in the world
  4. Call Press Pointer Key (Left Mouse Button) when the player pulls the trigger
  5. Call Release Pointer Key when the trigger is released
  6. Add a visible laser beam (line trace or Niagara beam) so the player can see where they are pointing
// VR Pawn — Trigger Pressed
Widget Interaction → Press Pointer Key
    Key: Left Mouse Button

// VR Pawn — Trigger Released
Widget Interaction → Release Pointer Key
    Key: Left Mouse Button

Best Practices for World-Space & XR UI

GuidelineWhy
Minimum font size 24ptWorld-space text must be readable at distance, especially in VR headsets with limited resolution
High contrast3D panels sit in varying lighting — use dark backgrounds with bright text
Large hit targetsVR pointing is imprecise — buttons should be at least 48×48 draw units
Billboard rotationFor floating labels, use “Look at Camera” so text is always readable
Interaction distanceSet a comfortable interaction range (1–3 meters) — too far feels disconnected, too close is hard to read
Tip: For VR, avoid screen-space UI entirely. All UI should be world-space Widget Components interacted with via the Widget Interaction Component. Screen-space overlays cause eye strain and break immersion in VR.

✓

Final Checklist

Before you call your UI complete, verify each element:

  • Widget Blueprint created with a Canvas Panel root and proper widget hierarchy
  • Anchors set on every widget so the UI works across different resolutions
  • Buttons styled with Normal, Hovered, and Pressed states
  • OnClicked events wired up in the Graph tab with correct logic
  • Common widgets used — Text, Image, Progress Bar, Slider, Check Box, Combo Box
  • Layout containers (Horizontal/Vertical Box) used instead of manual positioning where appropriate
  • HUD created with health bar and score text bound to gameplay variables
  • Pause menu with Resume, Settings, Quit — toggled with Escape, game paused, input mode switched
  • Background Blur used for pause overlay effect
  • UMG Animations added — fade in on construct, hover effects on buttons
  • Sound effects playing on button clicks (Play Sound 2D)
  • World-space UI set up via Widget Component for 3D panels
  • Widget Interaction configured for XR pointer interaction (if building for VR/AR)
  • Font sizes and contrast tested for readability at target viewing distance
